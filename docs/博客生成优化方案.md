# 博客生成系统质量升级方案（V2 · 精细执行版）

> 目标：在现有 orchestration + LangGraph 架构基础上，

>
> 本文档为 **Cursor 精确执行版规范**，强调：
> - 明确节点职责

# 一、现状问题诊断（必须理解）

当前流程：

```

fetch_and_extract_refs
→ propose_outline
→ interrupt(confirm)
→ write_sections
→ finalize_markdown

```

### 核心问题

1. 无“全文主线（thesis）”
2. 无“章节级强约束”
3. 引用机制弱（非结构化）
4. finalize 只是拼接，不是编辑
5. 无质量门禁
6. 300 字限制严重压缩信息密度

---

# 二、升级后的标准流程（V2 架构）

```

fetch_and_extract_refs
→ synthesize_refs              （新增）
→ plan_article                 （新增 · 核心）
→ propose_outline              （可合并）
→ interrupt(confirm_outline)
→ write_sections               （升级）
→ assemble_article             （替代 finalize）
→ quality_gate                 （新增）

````

---

# 三、数据结构标准（强制规范）

## 3.1 ref_cards 升级结构

当前：

```json
{
  "url": "...",
  "title": "...",
  "summary": "..."
}
````

升级为：

```json
{
  "ref_id": "r1",
  "url": "...",
  "title": "...",
  "summary": "...",
  "key_points": [
    {"kp_id": "r1.k1", "text": "..."},
    {"kp_id": "r1.k2", "text": "..."}
  ]
}
```

### 执行要求

* synthesize_refs 必须生成 key_points（3-6 条）
* 每条 key_point 不超过 80 字
* 严禁编造未出现在原文中的事实

---

## 3.2 plan_article 输出结构（核心）

必须输出：

```json
{
  "thesis": "全文核心论点（一句话）",
  "terminology": {
    "RAG": "检索增强生成"
  },
  "sections_plan": [
    {
      "id": "s1",
      "goal": "本节目标",
      "bullet_points": [
        "必须覆盖点1",
        "必须覆盖点2",
        "必须覆盖点3"
      ],
      "recommended_refs": ["r1.k1", "r2.k3"],
      "target_length": 1000
    }
  ]
}
```

### 强制规则

* bullet_points 至少 3 条
* 每节至少分配 1 个推荐引用
* target_length ≥ 800（专业报告）

# 四、Prompt 升级规范

---

## 4.1 大纲 Prompt 升级

新增要求：

```text
1. 每节必须有明确问题意识。
2. 每节 description 不得泛泛而谈。
3. 不得出现“首先、其次、最后”式模板语言。
```

---

## 4.2 plan_article Prompt（新增）

```text
你是资深编辑，请根据标题、参考资料和大纲，
构建完整写作蓝图。

必须输出 JSON：
- thesis
- terminology
- sections_plan

每节：
- 明确 goal
- 3-6 条 bullet_points
- 推荐引用（来自 key_points）
- 目标字数

禁止输出解释。
```

---

## 4.3 write_sections Prompt 升级（关键）

新增强约束：

```text
必须：
1. 覆盖所有 bullet_points。
2. 至少引用 1-2 条 recommended_refs。
3. 引用格式：[来源标题](URL)
4. 禁止编造数据。
5. 结构必须：
   - 背景
   - 分析
   - 论证
   - 小结
6. 字数目标 {target_length} ±15%
```

# 六、assemble_article（替代 finalize）

功能：

1. 添加一级标题
2. 自动生成引言（基于 thesis）
3. 自动生成总结（呼应 thesis）
4. 插入“参考资料”列表
5. 统一 Markdown 结构

禁止只做拼接。

# 七、质量门禁（Quality Gate）

新增节点：

## 7.1 基础规则

### 规则1：引用检查

```python
if final_md.count("](") < len(sections):
    trigger_rerun()
```

### 规则2：模板语言检测

禁止出现：

* “总之” 超过 2 次
* “显然”
* “值得注意的是” 超过 2 次

---

## 7.2 LLM 评估（可选）

Prompt：

```text
从以下维度打分：
1. 逻辑结构
2. 专业度
3. 信息密度
4. 引用真实性

1-10 分，低于 7 必须指出具体问题。
```

低于阈值 → rerun 低分 section。
